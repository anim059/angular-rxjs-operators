ShareReplay

What is the ShareReplay Operator?

The ShareReplay operator shares an observable’s subscription and replays the latest emitted values to new subscribers. Unlike share(), it caches the latest emitted values, ensuring that new subscribers receive the same previously emitted data without triggering a new execution. It has three parameters: bufferSize (default: Infinity) , windowTime (optional), scheduler (optional) – Allows scheduling the emissions. This operator is a specialization of replay that connects to a source observable and multicasts through a ReplaySubject constructed with the specified arguments. A successfully completed source will stay cached in the shareReplayed observable forever, but an errored source can be retried.

Common Use Cases:

  • Avoiding Duplicate API Requests: Ensures multiple subscribers get the same API response without making redundant HTTP calls.
  • Buffering Last Emissions for Late Subscribers : Useful for real-time streams where new subscribers need immediate access to previous values.
  • Keeping WebSocket Messages Available for New Subscribers : Prevents loss of recent messages when a new subscriber joins a WebSocket stream.
